3 Purpose, Goals and Audience

In creating this web-site, Jonas Konte wishes to reach out to the public and increase the overall awareness of the restaurant Capisco benissimo, and increase the potential for customer growth. This newly established enterprise wishes to offer customers the opportunity to dine in a rural location, that is not only close to downtown Trondheim, but also offers a variety of high quality meals inspired by his upbringing in Piemonte.

Our mission is to aid Jonas Konte in his desire to inform the public of this intriguing new restaurant, seldom seen in this segment. Combined with providing the visitor with conventional features like viewing the menu, booking a table, and a way of giving feedback. We will carefully design and structure our website so that it fits the needs of higher middle class families, which is the designated costumer group.

4 Navigational Structure

The page will be a SPA (Single Page Application) meaning the entire page will be viewed as a single index.html file. During the development process we will split each section of the page up into components. Each component can be developed independently (in seperate component.html files), and then compiled (or parsed on load) into the index file.

Each component will also have its own accompanying style.css file which will contain the style of the component. These will then be imported into the global.styles.css stylesheet that contain all styles for the page.

The naviational structure of the website is standard for SPA style websites. The navbar will be the main method of naviation. When the user clicks a link on the navbar, the browser should scroll the appropriate element into view. Using this very standard method of navigation means the user will be familiar with the navigational structure of the page only seconds after opening the page for the first time.

5 Page Layout and Appearance

When designing the page layout, we focused on the primary goal of getting costumers for the restaurant. We compiled an ordered list of importance for each component, and structured the page accordingly.

First impressions are important. That is why the landing page (header component) should be eye-catching and intriguing. A large image that takes up most of the initial viewport is a good way to catch the users attention. Around the sides of the image we will place a border to accomodate the navbar in its base position (non-sticky).

The first thing the user will see when scrolling is the address of the restaurant, and its opening hours, followed by the table reservation (booking) page. We chose to display the address and opening hours of the restaurant before the table reservation, because we believe people are more inclined to 'drop in' to the restaurant rather than reserve a table. (The restaurant should not normally be full enough to justify a reservation.) Displaying the table reservation high on the page is important to get customers to make reservation. When they visit the site, they are normally considering making a reservation but might change their mind if the reservation is too difficult to make.

After the booking page, the page contains the restaurants menu. The menu will by default be contracted, as not to occupy too much vertical space. The user may click on a category to expand the menu for this category.

As to the appearance of the page, the site will use Open Sans - a sleek, apple-like font. (The font used by this document.) We will also apply modern typographic design ideas, such as using weights to create contrast, to give the site a sleek, classy feel while maintaining easy readability. The color selection of the site will match the sleek typography by using an off-beige on dark-gray color-scheme with dark orange highlights.

The layout of the page will be somewhat responsive to change in viewport dimensions. All components will be designed to be scalable, but will not have defined different styles for different breakpoints. The navbar is the sole exception to this, as it will change to a dropdown menu when the veiwport can no longer reasonably fit all its links. In this small state the navbar will simply display the logo and a button when contracted.

6 Content

There will be 6 distinct components compiled into the index.html document:

  1. Navbar

    We have chosen to program the navbar in a separate HTML document, so we can easily manipulate it with javascript. The navbar will contain anchors for navigating up and down the site, and will be attached to the top of the page as the user scrolls down. The points it will navigate between are Menu, about us, book a table, contact. This is so that the user can easily find the desired information quickly.

  2. Header

    This is the first page the user will see when they enter the website. On this page, the logo should be eye-catching and it is repeated both at the top (in the margin) and on top of a slideshow of images. The images will set the mood of the restaurant and is located in the middle of the page with a margin on both sides, which will repeat down the entire website, and on the top. This page is to introduce the user to what the site represents and give the user an impression of what he / she can expect from the restaurant. This, in addition to the navbar, will give a simple and inviting first impression. The style of this page is repeated throughout the website. For a restaurant it is important that the user can easily find out what times the restaurant is open. Below the image, the user will find opening hours in style with the rest of the page, but with a background color that will give it a kind of "pop" so that it is easy to spot. The text will be centered.

  3. Booking

    This is the most important part of the page for the user and will be a form that allows the user to place a table order. The first line requests the user to provide at what date, time and how many poeple tha would like to eat. Firstly, there will be a form that asks what date they want to eat, here we povide the user withwo options, you can type in the date, or find it in the calender. The decision to include the calender is based on the desire to aid users who are booking a date further into future, chiefly in dertermening which day of the week the date is. Then, we ask the user to specicify when they would like to eat, followed by a form asking for the amount of guests. The next three lines only has one form per line, considering it requests information like name, phone number and e-mail, we ffound it natural to provide more space. It is also possible for the user to add any comments in a box below, in case they have any allergies or spesific needs. Once the user has placed an order, it is important that he / she receives a validation that it has been sent and received. A page will then appear and tell the user this.

  4. Menu

    This page will contain an overview of the restaurant's selection of dishes. The courses will be found in a pull-down list of different categories such as starter, main course, dessert etc. And then displayed as a classic menu with a fine line in the middle and courses on both sides of it. The name of the dishes are written in a red font and it's description in a light grey, we feel like this distinction will make the menu easier to navigate. A background color of soft beige is not only pleasant to look at, it also blends in well with the two font colors.

  5. About

    This should tell the user general info about the restaurant such as origin, idea behind, the founder and the food. It will contain headers of the topics, paragraphs and pictures.

  6. Contact

    This will contain the restaurants contact information. The text will be aligned in the center and written in white. Under this information there the restaurant's location on google maps will be shown, so that the user knows where it is. The position is given through google maps so that the user can retrieve the coordinates and efficiently get from a to b. This map will stretch over from right to left, but leave room for the black margin which is consistent throughout the whole page.
