4 Changes made and why

Map and contact changes

While working on the map and contact page the map felt a little small since the restaurants position on the map is so far off from the nearest city. Therefore it was decided to put the map and contact under eachother so it would be easier for the user to look at and use the map. It also made more sense to give the map a darker theme to match the rest of the page and not stick out like a sore thumb. The styling of the contacts got a little pop of color to not look so flat and tie it in with the general cholorscheme of the rest of the mainpage.

Menu changes

The menu remained mostly the same as in P2 with the exception of color changes for the fonts of the headers. Also we decided to increase the distance between the gray lines seperaiting categories on the menu to give it a more refined look.

6 Modularity

Imre was responsible for handling the modularity of the site. To acheive modularity we split up the page into seperate, independent components that we could each work on independently. This allowed us to work on the same page in parallell through GitHub. Imre set up a simple JavaScript framework for importing the components to the main index.html. Originally this framework would import a components markup aswell as its style, but we decided to instead use CSS import statements in the production build, to ensure the styles would be ready and loaded when the page loads. It also cleaned up the code in the developer console of chrome.
